# 项目开发规范

1. 不要重建项目，只在现有项目基础上修改。
2. 不要删除 content 文件夹读取逻辑。
3. 资料内容仍然以本地 content 文件夹为主。
4. Supabase 只用于登录、收藏、下载记录、用户行为数据。
5. 任何数据库变更，必须先在 supabase/ 文件夹中新建编号 SQL 文件，例如：
   - 001_profiles.sql
   - 002_favorites.sql
   - 003_downloads.sql
6. 不允许只口头提示“执行某某 SQL”，必须确保文件真实存在。
7. 每次修改前，先说明要改哪些文件。
8. 每次修改后，说明改了哪些文件、如何测试。
9. 每次新增功能，只做一个小功能，不要顺手开发无关功能。
10. 不要接微信公众号接口。
11. 不要接支付。
12. 不要把文章正文迁移到 Supabase。
13. 保持现有简洁、温和、大气的视觉风格。
